## Screenshot Guidelines

### Technical Specifications
- **Format**: PNG (preferred) or JPG
- **Resolution**: Minimum 1920x1080 (Full HD)
- **File Size**: Keep under 2MB each
- **Naming**: Use lowercase with hyphens (e.g., `admin-dashboard.png`)

### Capture Tips
1. **Use a clean browser**: Clear cache, close unnecessary tabs
2. **Full page**: Capture entire page or use scrolling screenshot tool
3. **Zoom level**: 100% (default browser zoom)
4. **Remove personal data**: Use demo accounts, not real data
5. **Good lighting**: If showing mobile, use good lighting
6. **Consistent styling**: Use same theme/mode for all screenshots

### Tools
- **Windows**: Snipping Tool, Snip & Sketch (Win + Shift + S)
- **Mac**: Screenshot tool (Cmd + Shift + 4)
- **Browser Extensions**: 
  - FireShot (full page screenshots)
  - Awesome Screenshot
  - Nimbus Screenshot
- **Online Tools**: 
  - Screely (add browser mockups)
  - MockUPhone (mobile mockups)

### Mockup Enhancement (Optional)
Use tools to add device frames:
- **Desktop**: https://screely.com/
- **Mobile**: https://mockuphone.com/
- **Multi-device**: https://smartmockups.com/

## After Capturing

1. **Review**: Check all screenshots are clear and readable
2. **Optimize**: Compress images if needed (use TinyPNG or similar)
3. **Organize**: Ensure filenames match those in README.md
4. **Commit**: Add to git repository
5. **Update README**: Ensure screenshot paths in README are correct

## Notes

- Screenshots show the **current state** of the application
- Use **demo data**, not production data
- Ensure all features are **visible and functional**
- Screenshots should be **recent** (not outdated UI)
- Consider adding **annotations** or highlights for key features (optional)
